# Telegram Bot на aiogram 3.24.0

Базовый скелет Telegram бота, созданный с использованием библиотеки [aiogram](https://docs.aiogram.dev/en/v3.24.0/).

## Описание

Это простой Telegram бот с базовым функционалом:
- Команда `/start` - приветствие пользователя
- Команда `/help` - справка по доступным командам
- Echo-обработчик - повторяет отправленные сообщения

## Установка

### Требования

- Python 3.10 или выше
- pip (менеджер пакетов Python)

### Шаги установки

1. **Клонируйте репозиторий** (если используете git):
   ```bash
   git clone <your-repo-url>
   cd 1_appka
   ```

2. **Создайте виртуальное окружение** (рекомендуется):
   ```bash
   python -m venv venv
   ```

3. **Активируйте виртуальное окружение**:
   
   На Windows:
   ```bash
   venv\Scripts\activate
   ```
   
   На Linux/Mac:
   ```bash
   source venv/bin/activate
   ```

4. **Установите зависимости**:
   ```bash
   pip install -r requirements.txt
   ```

5. **Настройте переменные окружения**:
   
   Скопируйте файл `.env.example` в `.env`:
   ```bash
   copy .env.example .env
   ```
   
   Откройте `.env` и вставьте токен вашего бота:
   ```
   BOT_TOKEN=your_bot_token_here
   ```
   
   **Как получить токен бота:**
   - Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
   - Отправьте команду `/newbot`
   - Следуйте инструкциям для создания нового бота
   - Скопируйте полученный токен в файл `.env`

## Запуск

Запустите бота командой:

```bash
python bot.py
```

Бот начнет работать в режиме long-polling (постоянно опрашивает серверы Telegram на наличие новых обновлений).

## Основные компоненты

### `bot.py`

Главный файл бота, содержащий:
- Инициализацию `Bot` и `Dispatcher`
- Обработчики команд (`/start`, `/help`)
- Echo-обработчик сообщений
- Функцию `main()` для запуска бота

### Обработчики

- **`command_start_handler`** - обрабатывает команду `/start`
- **`command_help_handler`** - обрабатывает команду `/help`
- **`echo_handler`** - повторяет любые отправленные сообщения

## Расширение функционала

Базовый скелет можно легко расширить:
- Добавить новые команды через `@dp.message(Command("command_name"))`
- Реализовать клавиатуры (inline и reply)
- Добавить Finite State Machine (FSM) для диалогов
- Использовать Router для модульной структуры
- Добавить middleware для логирования и обработки ошибок
- Интегрировать базу данных
